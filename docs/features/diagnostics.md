# Diagnostics & Comparison

## What it does

A small toolkit for answering "what's going on with this, and what changed?":

- **Flow timeline** — merge a resource's events and log activity into a single
  chronological view, so you can see the sequence of what happened rather than
  reading events and logs in separate places.
- **Resource diff** — compare two resources (or two versions of one) as a
  structured, field-level diff, making configuration drift and edits obvious.
- **RBAC access check** — ask whether a given actor can perform a given action on a
  resource, i.e. a built-in "can I?" checker for reasoning about permissions.

## Why it exists

Debugging Kubernetes is largely correlation work — lining up a symptom against a
timeline, a config change, or a missing permission. Each of these tools removes a
manual, error-prone step: stitching events and logs by hand, eyeballing two YAMLs
for differences, or shelling out to check access. Keeping them next to the resource
list means the answer is one click from the thing you're looking at.

## Notable constraints

- The flow timeline and diff are computed from data the API and metrics/log sources
  expose; they show what's retrievable, not a complete forensic record.
- The RBAC check reports what the API's access-review says is *allowed*; it reflects
  policy, not whether an action would otherwise succeed.
- All of these are read-only analyses — they explain state, they don't change it.

`pkg/k8s/diff.go`, `pkg/k8s/rbac.go`, and `frontend/src/features/diagnostics/`.
